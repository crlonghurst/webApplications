            Form Validation can be done in two places, Client Side and Server Side.
                It is best to do both.
            Why validate if we want users to have a simple time?
                1. To get the right data in the right format,
                2. To protect the user's data.
                3. To protect us.

            You can have several types of client-side 
                1. Built-in form Validation
                    By using the required attribute. Minlength and Maxlength. Min and max attributes, Type, Pattern.
                2. Javascript
                    by using the Constraint Validation API.
                    Applying this API will allow you to set the validation message which shows upwhen it's not validated a custom message appears.
            Fetch can have two parameters, usually just one,
              example with two parameters,
                const response = await fetch(url, {
                  method: 'POST',
                  mode: 'cors',
                  cache: 'no-cache',
                  credentials: 'same-origin',
                  headers: {
                    'Content-Type': 'application/json'
                  },
                  redirect: 'follow',
                  referrerPolicy: 'no-referrer',
                  body: JSON.stringify(data)
                });
                return response.json();

                Mode: "no-cors" allows a limited set of headers
                    Accept, Accept-language, Content-language, Content-Type,
